# UC降雨処理 面的可視化設計

## 1. 目的

本書は、UC 降雨処理 GUI に追加する面的可視化ビューの設計を定義する。

本機能の目的は以下とする。

- ある時刻またはある累加時間幅におけるセル分布を面的に把握できること
- 流域境界とセル分布を同時に確認できること
- 面ビューからセル単位グラフへ自然に遷移できること
- AI テストモードからも面ビューを操作・評価できること


## 2. スコープ

初期設計の対象は以下とする。

- 単一時刻の 1 時間雨量ヒートマップ
- 指定指標の累加雨量ヒートマップ
  - `1h`
  - `3h`
  - `6h`
  - `12h`
  - `24h`
  - `48h`
- 流域ポリゴン境界の重ね表示
- セルクリックによる `polygon_local_row / polygon_local_col` 選択
- AI テストモード用の状態・画像・操作入出力

初期設計の対象外:

- 背景地図タイル表示
- Web GIS 連携
- 時間アニメーション
- 3D 表示


## 3. 採用方針

### 3.1 表示方式

表示は Tkinter へ埋め込んだ `matplotlib` を第一候補とする。

理由:

- 既存依存で実装可能
- 既存のグラフ描画基盤と親和性が高い
- カラーバー、ポリゴン境界、クリックイベントを扱いやすい


### 3.2 面ビューの位置づけ

面ビューはグラフ出力の代替ではなく、セル選択と状況把握の補助である。

基本フロー:

1. 流域と対象時刻または指標を選ぶ
2. 面ビューでセル分布を見る
3. クリックしたセルを候補セルとして選ぶ
4. 必要に応じてグラフ出力へ進む


## 4. 表示モード

### 4.1 単一時刻ビュー

表示内容:

- 指定時刻の `rainfall_mm`
- 流域ポリゴン境界
- セル境界または塗りつぶしセル
- カラーバー

用途:

- 降雨分布の瞬間把握
- 入力データの妥当性確認


### 4.2 累加雨量ビュー

表示内容:

- 指定時刻における rolling 累加値
- 指標 `1h/3h/6h/12h/24h/48h`
- 流域ポリゴン境界
- カラーバー

用途:

- イベント時の分布把握
- グラフで見たピークの面的確認


### 4.3 流域集計補助ビュー

初期実装では、流域集計値そのものを面に描くのではなく、元セル値を面表示する。

理由:

- 流域集計は 1 本の時系列であり面に戻す意味が薄い
- まずはセル分布を見せるほうが判断しやすい


## 5. 画面構成案

GUI 内では以下の 2 案が考えられる。

### 案 A: 候補セルエリアの上部に面ビューを追加

- 上: 面ビュー
- 下: 候補セルテーブル
- 右下または別枠: 実行ログ

利点:

- 候補セルと面の対応が近い
- 現行 GUI に載せやすい


### 案 B: Notebook で `表ビュー` と `面ビュー` を分ける

- タブ1: 候補セルテーブル
- タブ2: 面ビュー

利点:

- 画面密度を抑えやすい
- 面ビューの領域を大きく取れる

初期実装の第一候補は案 B とする。

理由:

- 現在の GUI は既に情報量が多い
- 面ビューは広い表示領域が必要
- AI テストでもタブ単位の状態把握がしやすい


## 6. 入力項目

面ビュー専用に最低限必要な入力は以下とする。

- 流域名
- データセット優先条件
  - 任意
- 表示時刻
- 表示モード
  - `単一時刻`
  - `累加雨量`
- 指標
  - 累加雨量時のみ
- 色スケール方式
  - `固定`
  - `自動`

将来拡張:

- カラーマップ選択
- 最小値 / 最大値固定
- `log` スケール


## 7. データ取得設計

面ビュー用に必要なデータは DB から取得する。

### 7.1 単一時刻ビュー

取得対象:

- 指定時刻の `cell_timeseries`
- 対象流域の `polygon_cell_map`
- `polygons.geometry_wkt`

### 7.2 累加雨量ビュー

取得対象:

- 指定時刻までの必要期間の `cell_timeseries`
- rolling 計算に必要な過去データ
- 対象流域の `polygon_cell_map`
- `polygons.geometry_wkt`

計算:

- 既存の rolling ロジックを流用する
- 面ビュー用にセル単位で値を計算する


## 8. 描画設計

### 8.1 セル描画

セルは矩形ポリゴンとして描画する。

必要要素:

- `x_center`
- `y_center`
- `cell_width`
- `cell_height`

矩形ポリゴンを生成し、値に応じて塗り分ける。


### 8.2 ポリゴン境界

流域ポリゴン境界は黒線または濃い灰色線で重ねる。

方針:

- セル塗りを邪魔しない線幅
- 輪郭が分かる程度の濃さ


### 8.3 カラーバー

カラーバーは必須とする。

表示内容:

- 指標名
- 単位
  - `mm`


### 8.4 欠測表現

欠測セルは灰色またはハッチングで表示する。

理由:

- 値 0 と欠測を区別するため


## 9. インタラクション設計

### 9.1 セルクリック

面ビュー上でセルクリックしたとき:

- 対応する `polygon_local_row`
- `polygon_local_col`
を GUI 入力欄へ反映する

補足:

- 対象セルは強調表示する
- 候補セルテーブルがある場合は該当行も選択する


### 9.2 時刻変更

時刻入力変更時:

- 面ビューの再描画ボタンを有効にする
- 自動再描画は初期実装では必須としない


### 9.3 グラフとの連携

面ビューでセルを選択した後、

- `グラフ出力`
を押せばそのセルで既存の時系列グラフを出せること


## 10. AI テストモード設計

AI が面ビューを操作・評価できるように、以下を追加する。

### 10.1 widget_id

最低限以下を持つ。

- `tab.spatial_view`
- `canvas.spatial_view`
- `combobox.spatial_mode`
- `entry.spatial_timestamp`
- `combobox.spatial_metric`
- `button.render_spatial_view`
- `button.clear_spatial_selection`


### 10.2 widget tree 拡張

`widget_tree` には以下を追加する。

- 面ビュー領域サイズ
- 現在表示しているモード
- 現在表示している時刻
- 選択セル
- カラーバー有無
- 画像保存先


### 10.3 操作要求

AI テストモードでは以下の操作を許可する。

- `click`
- `set_text`
- `select`
- `render_spatial_view`
- `click_canvas_point`

`click_canvas_point` は

- 画面座標
または
- データ座標
を受けてセル選択を行う


### 10.4 テスト結果

AI が UX を評価できるよう、以下を保存する。

- 面ビュースクリーンショット
- 選択セル情報
- 画面上の警告文
- 最後に描画したデータ件数

AI が確認したい観点:

- ボタンが動作するか
- 時刻変更で面が変わるか
- ポリゴン境界が見えるか
- 文字切れがないか
- カラーバーが読めるか
- クリックしたセルが期待どおり選択されるか


## 11. 追加サービス案

面的可視化を GUI へ入れる前に、サービス層へ以下を追加するのが望ましい。

- `build_spatial_view_frame(...)`
  - 指定時刻のセル値一覧を返す
- `build_spatial_metric_frame(...)`
  - rolling 指標のセル値一覧を返す
- `build_cell_polygons(...)`
  - 描画用セル矩形を返す

理由:

- GUI から直接 SQL と描画前加工を持ちすぎないため
- AI テストでもロジックと表示を分離しやすいため


## 12. 実装順

1. サービス層に面ビュー用取得関数を追加
2. GUI に `面ビュー` タブを追加
3. 単一時刻ビューを実装
4. 累加雨量ビューを実装
5. セルクリック選択を実装
6. AI テストモードの `canvas` 操作を実装
7. スクリーンショットと widget tree の出力を拡張


## 13. 検証項目

最低限の確認:

1. 指定時刻の 1 時間雨量が面表示される
2. 指定時刻の 24h 累加が面表示される
3. 流域境界が重なる
4. セルクリックで `local_row/local_col` が入る
5. 面ビューからグラフ出力へつながる
6. AI テストモードで面ビュータブを開ける
7. AI テストモードでセルクリック相当操作ができる
8. 面ビューのスクリーンショットで文字切れや重なりを確認できる
