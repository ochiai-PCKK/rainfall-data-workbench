# UC Rainfall ZIP Flow テスト仕様

## 1. 目的

本書は `uc_rainfall_zipflow` のテスト観点を整理し、`run_uc_rainfall_gui.py` から起動する GUI を含む
機能の検証範囲と期待結果を明確にする。

目的は次のとおり。

- 実装の回帰を早期に検出する
- GUI と CLI の責務を分けて検証する
- 解析雨量モードと Excel モードの差分を明文化する
- 将来の `pytest` 化にそのまま落とし込める粒度にする

## 2. テスト方針

### 2.1 テスト階層

テストは次の3階層に分ける。

1. 単体テスト
   - 純粋関数や変換ロジックを対象にする
   - ファイル I/O や GUI 依存をできるだけ避ける
2. 結合テスト
   - `run_zipflow` / `run_excel_mode` / GUI からの呼び出し経路を対象にする
   - 一時ディレクトリと小さなサンプルデータを使う
3. GUI テスト
   - `run_uc_rainfall_gui.py` で起動する画面の初期状態、状態復元、例外処理を対象にする
   - 自動化できるものは GUI テストモードまたは Tk の状態参照で確認する

### 2.2 優先順位

- P0
  - 起動不能
  - 期間計算誤り
  - 出力先誤り
  - 例外が GUI から漏れる問題
- P1
  - 状態復元不備
  - 候補一覧の不整合
  - Excel / 解析雨量モードの分岐誤り
- P2
  - 表示文言の揺れ
  - 軽微な補助動作

### 2.3 テストデータ方針

- 可能な限り小さい入力を使う
- ZIP / Excel は固定のサンプルを用意する
- 壊れた Excel、候補なし、期間不足などの異常系サンプルも用意する
- 出力先は一時ディレクトリを使い、既存成果物を汚さない

## 4. テスト観点

### 4.1 起動

- `run_uc_rainfall_gui.py` から GUI が起動する
- 初期状態でウィンドウが表示される
- 解析雨量モードが初期選択になっている
- 既存設定がある場合は復元される

### 4.2 モード切替

- 解析雨量モードから Excel モードへ切り替えられる
- Excel モードでは固定項目が表示/非表示切替される
- モード切替で前モードの設定が破壊されない

### 4.3 解析雨量モード

- ZIP ディレクトリ未指定では候補日が空になる
- 候補更新で ZIP 群から対象日が列挙される
- 手動期間指定は 3日 / 5日 のみ許可される
- 自動期間指定では対象日選択から実行設定が組まれる
- `3d_left` / `3d_center` / `3d_right` の期間解釈が要件どおりである
- 参考グラフの基準日が寄せ方向に応じて補正される

### 4.4 Excel モード

- Excel ファイル未指定では候補一覧を作れない
- シート名からイベント候補が抽出される
- 再分割シートが候補として分離される
- 選択中イベントを使ってスタイル調整プレビューが作成される
- 選択がない場合はテンプレートプレビューへフォールバックする

### 4.5 状態保存 / 復元

- 保存した入力パスが戻る
- 候補一覧が再生成される
- 選択済みの候補日 / シートが復元される
- 出力種別や流域選択が意図せず変わらない

### 4.6 出力

- 出力先ディレクトリが仕様どおり作られる
- 参考グラフのファイル名が基準日と span を含む
- 出力衝突時の動作が `cancel` / `overwrite` / `rename` のいずれかで制御される

### 4.7 エラー処理

- 壊れた Excel でスタイル調整を押しても GUI が落ちない
- 候補ゼロ、入力不足、期間不正で適切なエラー文言が出る
- バックエンド例外が Tk の未処理例外として漏れない

## 6. 自動化対象

自動化優先度は次のとおり。

- 最優先
  - 期間計算
  - 参考グラフ基準日
  - Excel 候補抽出
  - 状態復元
  - 例外時に GUI が落ちないこと
- 次点
  - 出力ファイル名
  - 衝突時の挙動
  - モード切替の表示差分
- 手動中心
  - 画面レイアウト
  - 文字詰まり
  - 見た目の細部

## 7. 必要なテストフィクスチャ

- 正常 ZIP サンプル
- 欠損 ZIP サンプル
- 正常 Excel サンプル
- 壊れた Excel サンプル
- 状態保存ファイルのサンプル
- 一時出力先

## 8. 受け入れ基準

- P0 のテストケースがすべて通る
- GUI 起動経路で未処理例外が出ない
- 解析雨量モードと Excel モードの差分がドキュメント化されている
- 今後の `pytest` 実装時に、ケース名をそのままテスト名へ流用できる
