# UC Rainfall ZIP Flow Overview

## 1. 背景

現行の `uc_rainfall` は、DB 蓄積を前提にした解析基盤として設計されている。  
一方で、大量 ZIP を順次処理するバッチ用途では、DB 容量肥大化や運用コストが課題になる。

このため、DB を介さずに ZIP から直接グラフを作成する別サブパッケージを用意する。


## 2. 目的

`uc_rainfall_zipflow` は、以下を目的とする。

- ZIP 群を期間指定で走査する
- ZIP 単位で時系列を抽出する
- 流域判定と系列生成を行う
- 指定指標の最大イベントグラフを出力する
- DB を使わずにバッチ処理を完結させる


## 3. スコープ

本サブパッケージの対象は以下とする。

- 入力: ZIP ディレクトリ、流域ポリゴン、期間条件
- 処理: 時刻復元、流域セル抽出、系列作成、イベント抽出
- 出力: PNG グラフ、実行ログ（CSV/JSON）

本サブパッケージの対象外は以下とする。

- 長期保存を目的とした DB 正規化
- API 提供基盤
- GUI での対話操作


## 4. 想定ユースケース

- 3日ZIPが大量にある期間をまとめて処理したい
- DB 容量を増やさずに、まずグラフ成果物を得たい
- 後続のレビュー用に、出力と実行ログを残したい

## 6. 想定 CLI

コマンド名（案）:

- `uc-rainfall-zip`

サブコマンド（案）:

- `plan`（対象ZIP確認）
- `run`（一括グラフ出力）
- `plot`（単体ZIP検証）
- `report`（実行結果サマリ）


## 7. 高レベル処理フロー

1. ZIP ディレクトリを走査し、期間条件で対象ZIPを確定する  
2. ZIP を 1件ずつ解析し、観測時刻とセル雨量を抽出する  
3. 流域ポリゴンと照合して対象セルを確定する  
4. `cell/sum/mean/weighted` 系列を作成する  
5. 指標別に最大イベントを抽出してグラフ出力する  
6. 実行ログと出力一覧を保存する


## 8. 期待効果

- DB 容量増加の抑制
- 大量 ZIP の単純なバッチ処理に最適化
- 実行経路が短く、障害切り分けが容易
- 現行 `uc_rainfall`（DB基盤）との責務分離
