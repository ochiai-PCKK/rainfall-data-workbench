# 流域雨量グラフ GUI設計

## 1. 目的

- `uc_rainfall_zipflow` の CLI 機能を、運用者が迷わず実行できる GUI で提供する。
- 既定運用は `nishiyoke_higashiyoke` 出力を中心とし、詳細設定は必要時のみ表示する。
- AI テスト（操作自動化）しやすい、状態が明確な UI を定義する。

## 1.1 参照ワイヤーフレーム

- 最新ワイヤーフレーム: `docs/uc_rainfall_zipflow/uc_rainfall_zipflow_gui_wireframe.pen`

## 2. 前提・制約

- 必須ポリゴンは `西除川流域` と `東除川流域`。
- `西除川+東除川` は常に生成可能で、既定の対象流域とする。
- `大和川流域界` は任意（存在時のみ選択可能）。
- 時刻は JST 基準で扱う。

## 3. 画面構成

### 3.1 レイアウト

- ヘッダー: 画面名、対象流域チップ、JST表示
- 左ペイン: モード別設定パネル（解析雨量 / Excel）
- 右ペイン: 実行状況（実行ログ / 生成物サマリ / グラフスタイル調整）
- 下部ステータス: 実行状態（待機・実行中・完了・失敗）

### 3.1.1 実装構成（GUI分割）

- `gui/app.py`: 共通シェル（ヘッダー、実行、ログ、出力一覧、状態管理）
- `gui/rain_mode_panel.py`: 解析雨量モード専用フォーム
- `gui/excel_mode_panel.py`: Excelモード専用フォーム
- `gui/style_tuner_window.py`: スタイル調整ウィンドウ
- `gui/types.py`: モード共通の受け渡し型
- `style_tuner_core.py`: スタイル調整入力の正規化（CSV/Excel実データ/テンプレート）

### 3.2 グループ

1. 解析雨量モード（`rain_mode_panel`）
- 期間指定（開始日 / 終了日）
  - 制約: 期間差は 3日 または 5日 のみ
- 入力ZIPディレクトリ（参照ボタンあり）
- ポリゴンディレクトリ（参照ボタンあり）
- 出力ディレクトリ（参照ボタンあり）
- ログ出力有効化（チェック）

2. Excelモード（`excel_mode_panel`）
- 入力Excelファイル（参照ボタンあり）
- イベント候補一覧（シート名）
  - `YYYY.MM.DD` と `【再分割】YYYY.MM.DD`
  - 複数選択可能
- グラフ期間（3日 / 5日）選択
- 出力ディレクトリ（参照ボタンあり）
- ログ出力有効化（チェック）

3. 実行条件（共通）
- 対象流域（既定: `nishiyoke_higashiyoke`）
- 出力種別（GUI表示: `raster`, `raster_bbox`, `整形時系列グラフ(plots_ref)`, `分析CSVセット(analysis_csv)`）
- `plots` はGUI非公開（互換維持のためCLIのみ利用可）
- グラフ種別（`sum`, `mean`）

4. 実行
- 実行ボタン（主ボタン）
- 停止ボタン（将来拡張。初期実装では非表示）

5. グラフスタイル調整（右ペイン）
- 対象CSV（自動選択 / 手動選択）
- 自動選択ルールの補足表示
- 表示期間（3日/5日）
- グラフスタイル調整ボタン
- 設定反映ボタン

## 4. ユーザーに見せるもの・見せないもの

### 4.1 見せる（運用に必要）

- 実行に必要な最小入力
- 進捗ログ（採用ZIP数、対象期間、出力先）
- 成果物の出力先パス

### 4.2 通常は見せない（詳細設定へ）

- `window-mode` と関連パラメータ
- `style-profile` の直接指定
- デバッグ用内部値（coverage閾値、内部補正値など）

## 5. 入力バリデーション

- 必須項目未入力は実行不可。
- 開始日・終了日は `YYYY-MM-DD` のみ受理。
- `start <= end` を必須。
- 期間差は 3日 または 5日 のみ許可。
- 流域選択に `yamatogawa` が含まれる場合、該当ポリゴン存在を事前確認。

## 6. 実行フロー

1. モード別パネルから `ModePayload` を取得
2. `app.py` で `RunConfig` 相当へ変換
3. 事前検証（パス・日付・選択値）
4. 実処理呼び出し（`run_zipflow` / Excelモード処理）
5. 結果表示（成功: 出力先、失敗: エラーメッセージ）
6. ログ・前回設定を保存

## 7. ログとメッセージ

- 表示文言は日本語で統一する。
- 実行ログは「時刻 + レベル + メッセージ」の1行形式。
- ユーザー向けエラーは原因と対処を短文で表示する。

## 7.1 表示名の日本語フォールバック

- GUIのラベル、ボタン、選択肢は日本語表示を既定とする。
- 内部処理では既存の英語キー（例: `plots_ref`, `sum`, `mean`, `nishiyoke_higashiyoke`）を維持する。
- UI層で `表示名 <-> 内部値` マッピングを持ち、実行時は内部値へ変換して `RunConfig` を生成する。
- マッピング未定義値が来た場合は、内部値をそのまま表示して処理継続する（フォールバック）。
- 例:
  - `plots_ref` -> `整形時系列グラフ`
  - `analysis_csv` -> `分析CSVセット`
  - `sum` -> `重み付き合計`
  - `mean` -> `流域平均`
- `analysis_csv` 選択時の実出力先は `analysis_csv/` ディレクトリとする。

## 8. 設定キャッシュ

- 保存先: `config/uc_rainfall_zipflow/gui_state.json`
- 最低保持項目:
  - `input_zipdir`
  - `output_dir`
  - `period_start`
  - `period_end`
  - `selected_regions`
  - `selected_outputs`
  - `ref_graph_kinds`
  - `style_profile_path`

## 9. AIテスト容易性

- 主要ウィジェットに安定ID（`name`）を付与する。
- ボタン押下で結果が必ずログに残る設計にする。
- モーダル多用を避け、状態は右ペインログで追跡可能にする。
- 失敗ケース（入力不足、範囲不正）を再現できるテストシナリオを用意する。

## 10. 初期実装スコープ

- 対象: 単一ジョブ実行GUI（バッチ管理は対象外）
- 必須機能:
  - 実行設定入力
  - `run_zipflow` 実行
  - 実行ログ表示
  - 設定保存/復元
- 将来拡張:
  - ジョブキュー
  - 実行中断
  - 出力プレビュー統合

## 11. グラフスタイル調整連携

### 11.1 連携方針

- `style-profile` のファイル連携だけでなく、GUIからチューナーを直接起動できるようにする。
- 本体GUIの期間（3日/5日）と整合するプレビュー期間をチューナーへ引き継ぐ。

### 11.2 追加UI

- 本体GUIに「グラフスタイル調整」ボタンを追加する。
- ボタンは `plots_ref` 利用時に主導線として表示する。

### 11.3 起動時の引き継ぎ

- 本体GUI -> チューナーへ次を引き継ぐ:
  - `frame`（利用可能なら）
  - `source_kind`（`excel` / `csv` / `template`）
  - `value-kind`（`sum` / `mean`）
  - `preview-span`（`3d` / `5d`）
  - `profile-path`
- `input-csv` の選択優先順位:
  - 解析雨量モード（後方互換）:
  1. 現在GUIで選択中の流域に対応する最新 `*_timeseries.csv`
  2. 直近実行結果で作成された `*_timeseries.csv`
  3. 見つからない場合はファイル選択ダイアログで手動指定
  - Excelモード（推奨経路）:
  1. 選択中イベントの先頭1件から生成した実データ
  2. 候補未選択時はテンプレート
- GUI上に「対象CSVの使い方」説明を表示する:
  - 分析CSV出力あり: そのCSVで実データの見た目調整を行う
  - 分析CSV未出力: テンプレート（疑似データ）で調整を行う

### 11.4 保存後の同期

- チューナー側でプロファイル保存後、本体GUIで設定反映できる導線を設ける。
- 少なくとも「設定反映」ボタンを本体GUI側に置き、保存直後に反映可能とする。
- チューナーの保存は次の2段構えにする:
  - 主導線: 既定先へ上書き保存（`config/uc_rainfall_zipflow/styles/default.json`）
  - 詳細操作: 名前を付けて保存（任意パス）

### 11.5 制約

- 3日データしかないCSVではチューナー側 `5d` を無効化する（既存仕様）。
- 本体GUI側も同一制約を前提に表示する。
