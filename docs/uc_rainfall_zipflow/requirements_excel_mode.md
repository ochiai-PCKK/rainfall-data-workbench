# UC Rainfall ZIP Flow Excelモード 要件定義

## 1. 文書の目的

本書は `uc_rainfall_zipflow` の Excelモード（流域平均雨量の既集計データ入力）の要件を定義する。  
対象入力は `data/excel_input/ティーセン分割_面積・ティーセン係数.xlsx` と同形式の `.xlsx` とする。

## 2. 前提

- イベント日付シートには、既に流域平均雨量（時間雨量）が計算済みである
- ZIPモードで実施しているポリゴン重ね合わせ・空間重み計算は実施しない
- 本モードは `plot_ref` の作成に限定する
- 時刻基準は JST とする

## 3. スコープ

### 3.1 対象

- Excelブックのイベント日付シートの読取
- 時系列の正規化（`plot_ref` 入力用）
- `plot_ref` グラフ出力

### 3.2 対象外

- `raster` / `raster_bbox` 出力
- ポリゴン由来の `analysis_csv`（セル別CSV）出力
- ポリゴン入力を前提とした空間演算

## 4. 入力要件

- 入力は `.xlsx` ファイル1件を指定する
- 対象シートは以下とする
  - 通常イベントシート: `YYYY.MM.DD`
  - 再分割イベントシート: `【再分割】YYYY.MM.DD`
- 管理シート（例: `ティーセン係数`, `降雨継続時間`, `確率評価`）は処理対象外とする
- シート名から日付を解釈できない場合は対象外として無視する

## 5. シート選択要件

- 同一日付に通常シートと再分割シートが両方ある場合も、統合せず別候補として扱う
- したがって、`YYYY.MM.DD` と `【再分割】YYYY.MM.DD` は独立に選択可能とする
- ユーザーは同一日付の通常/再分割を両方選択して実行できる
- ログには選択されたシート名をそのまま出力する

## 6. データ抽出要件

- 時刻列は `B列` を使用する
- 主要系列は `Q列`（流域平均雨量の時間雨量）を使用する
- `R,S,T,U,V列`（3/6/12/24/48時間雨量）は参照可能列として扱う
- 時刻は Excel の表示文字列ではなくセル値（datetime）を正とする

## 7. 時間正規化要件

- 各イベントは 1時間間隔で連続していること
- 各イベントの点数は 120 点（5日 x 24時間）固定とする
- `base_date` はシート名日付とし、時系列は `base_date ±2日` の 120 点に一致すること
- 120点不一致または1時間間隔不一致の場合はエラー終了とする

## 8. 品質前提（厳格モード）

- `Q列` は欠損なしを前提とする
- `Q列` 欠損、型不正、時刻重複、時刻逆転、点数不一致は補完せずエラー終了とする
- 一部イベントだけ不正でも、既定動作は全体中断とする

## 9. 出力要件

- 出力対象は `plot_ref` グラフのみとする
- 出力先は `output_root/plots_reference/` とする（`base_date` 階層は作らない）
- 出力ファイル名は以下とする
  - `{region_key}_{base_YYYYMMDD}_{span}_{sum|mean}_overview.{png|svg}`
- PNG 出力を必須とし、SVG 出力はオプションで許可する
- 既存ファイル衝突時は `cancel` / `overwrite` / `rename` で制御する
- `rename` は `_v2`, `_v3` の連番付与とする

## 10. ログ要件

- 実行ログには少なくとも以下を記録する
  - 入力Excelパス
  - 採用シート一覧
  - 日付ごとの採用理由（再分割優先 / 通常採用）
  - 入力検証結果（120点・1時間間隔・Q列検証）
  - 出力件数と衝突時挙動

## 11. インターフェース要件（最小）

- Excelモードではポリゴン指定を要求しない
- Excelモードで `raster` / `raster_bbox` / `analysis_csv` が指定された場合はエラー終了とする
- エラーメッセージは不足項目と修正方法が分かる文言で返す
- Excelモードでは、Excel内から抽出したイベント候補（シート名）をGUIで一覧表示する
- イベント候補は複数選択可能とする
- グラフ期間は GUI で以下を明示選択可能とする
  - `5日`
  - `3日前寄せ`（`base_date-2日` 〜 `base_date`）
  - `3日中央`（`base_date-1日` 〜 `base_date+1日`）
  - `3日後寄せ`（`base_date` 〜 `base_date+2日`）
- 複数イベント選択時は、選択した全イベントについて順次 `plot_ref` を生成する
- スタイル調整は、選択中イベントのExcel実データを優先してプレビュー入力に使う
- 複数イベントが選択されている場合、スタイル調整に使う実データは先頭1件のみとする
- Excelモードでは CSV 指定・CSV 自動探索は使用しない
- Excelモードでイベント未選択のときのみテンプレートデータでプレビュー起動を許可する

## 12. 受け入れ条件

- Excel入力のみで `plot_ref` グラフを生成できる
- 3日選択時に `前寄せ/中央/後寄せ` の3通りで切替出力できる
- 同一日付で再分割シートがある場合、再分割シートが採用される
- `Q列` 欠損や120点不一致時に理由付きでエラー終了する
- 出力衝突時に `cancel` / `overwrite` / `rename` が選択どおり動作する
- `raster` / `raster_bbox` / `analysis_csv` は生成されない
