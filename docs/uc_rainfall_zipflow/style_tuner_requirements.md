# UC Rainfall ZIP Flow グラフスタイル調整 要件定義

## 1. 目的

`plots_ref` の体裁（余白、線幅、フォント、テーブル位置など）を、  
ユーザーが GUI で数値調整し、ほぼ即時にプレビュー反映できる仕組みを定義する。

## 2. スコープ

### 2.1 対象

- 体裁調整専用 GUI（本番処理画面とは分離）
- 既計算の時系列 CSV を入力にした高速プレビュー
- スタイル設定の保存/読込（JSON）
- 保存したスタイルを `uc_rainfall_zipflow` 本番出力へ適用

### 2.2 対象外

- ZIP 展開・空間計算・重み計算そのものの再実行最適化
- Web UI 化
- 複数ユーザー同時編集

## 3. ユーザーストーリー

- 利用者は、スライダーや入力欄で余白や線幅を変えると、即時に見た目を確認できる。
- 利用者は、気に入った体裁をプロファイルとして保存し、次回再利用できる。
- 利用者は、CLI 実行時に保存済みプロファイルを指定して同じ見た目で出力できる。

## 4. 機能要件

### 4.1 プレビュー入力

- 入力は `*_timeseries.csv`（必須）とし、ZIP再読込は行わない。
- CSV から `observed_at_jst`, `weighted_sum_mm`, `weighted_mean_mm` を読み込む。

### 4.2 調整可能パラメータ

- 図サイズ: 幅/高さ（必須）
- 出力解像度: dpi（PNG向け）
- 余白: left/right/top/bottom, hspace
- 棒: 幅、色、エッジ線幅
- 折れ線: 色、線幅
- 軸: 目盛フォントサイズ、タイトルフォントサイズ
- テーブル: 高さ、1行目Y、2行目Y、縦線線幅

図サイズ仕様:

- 内部パラメータは inch で保持する
- GUI 表示は px 併記を許可する（換算: px = inch × dpi）
- PNG 出力は幅/高さ + dpi を反映する
- SVG 出力は幅/高さを反映し、dpi は実質無視される前提で扱う
- プレビューと本番出力で同一サイズ再現を必須とする

### 4.3 反映方式

- 値変更時は同一 Figure/Axes を再利用して再描画する。
- 再描画は 500ms 以内（目標）とする。

### 4.4 プロファイル保存

- 保存形式は JSON。
- プロファイルは複数保存可能（名前付き）。
- デフォルトプロファイルを 1 件持つ。
- サイズ関連パラメータ（`fig_width`, `fig_height`, `dpi`）を保存対象に含める。
- 標準保存先は `config/uc_rainfall_zipflow/styles/default.json` とする。

### 4.5 本番適用

- CLI に `--style-profile PATH` を追加し、`plots_ref` 出力へ適用する。
- 指定なしは現行デフォルト体裁を用いる。

### 4.6 MVP 優先実装項目

初期実装（MVP）では、以下の調整項目を優先対応とする。

- サイズ: `fig_width`, `fig_height`, `dpi`
- 余白: `left`, `right`, `top`, `bottom`, `hspace`
- フォントサイズ: タイトル、軸ラベル、目盛
- 線幅: 折れ線、棒エッジ、テーブル縦線
- グリッド: 表示ON/OFF、線幅、色
- テーブル: 高さ、1行目Y、2行目Y

## 5. 非機能要件

- UI 文言・ログは日本語。
- プレビュー操作時にアプリがフリーズしない。
- 不正プロファイル読込時はエラー内容を明示してデフォルトへフォールバック可能。

## 6. 受け入れ条件

- GUI で最低 10 項目以上の体裁が即時調整できる。
- プロファイル保存→再起動→再読込で同一見た目を再現できる。
- CLI `--style-profile` 指定で GUI プレビューと同じ体裁が出力される。
- 既存 CLI（プロファイル未指定）は後方互換で動作する。
