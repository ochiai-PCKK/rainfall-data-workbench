# Changelog

このプロジェクトの変更履歴はこのファイルで管理します。  
形式は [Keep a Changelog](https://keepachangelog.com/ja/1.1.0/)、バージョニングは SemVer を前提にします。

## 運用ルール

- バージョンの基準は `pyproject.toml` の `[project].version`。
- 変更はまず `Unreleased` に追記。
- `pyproject.toml` の version を更新したタイミングで、`Unreleased` を同じバージョン見出しへ確定（リリース日付を付与）。
- セクションは原則 `Added` / `Changed` / `Fixed` を使用。

## [0.3.0] - 2026-03-27

### Added

- 実行エンジン切替を追加。  
  `cli run --engine` で `python` / `rust_pyo3` を選択可能にした。
- 集計コア比較用の `benchmark` サブコマンドを追加。  
  Python / Rust / （任意で）PyO3 の速度・メモリ・精度比較を出力できるようにした。
- Rust 実装を追加。  
  `rust/weighted_core`（ネイティブベンチ用）と `rust/weighted_core_pyo3`（Python拡張）を導入。
- PyO3 拡張ビルド用スクリプト `scripts/pyo3_wheel.ps1` を追加。
- 解析雨量グラフの中間データJSON出力を追加。  
  `plots_reference/_intermediate.json` を出力し、後段の再描画に利用できるようにした。
- 開発者ツールに「解析雨量: 中間JSONからグラフ再出力」を追加。  
  中間JSONから再描画し、PNGの2x4マージ画像を生成できるようにした。

### Changed

- 複数対象日の参照グラフ（`plots_ref`）を、共通軸上限を算出して一貫したスケールで描画する方式に変更。
- 軸上限計算・軸設定ロジックを見直し、スタイル調整時の挙動を改善。
- リリース用 GitHub Actions（`.github/workflows/release.yml`）を強化。  
  `uv sync` / PyInstaller / `Compress-Archive` にリトライ処理を追加し、失敗時の回復性を高めた。

### Fixed

- 参照グラフの軸アラインメント不整合を修正。  
  単体テスト（`test_axis_alignment`）を追加して再発防止を行った。

## [0.2.0] - 2026-03-18

### Added

- CLI に `excel-candidates` サブコマンドを追加。  
  入力Excelから候補日CSVを2点出力できるようにした。
  - `<Excel名>_候補日付リスト.csv`（日付・シート名・再分割フラグ）
  - `<Excel名>_候補日付一覧_unique.csv`（重複除去日付）
- Excel候補抽出ロジックを `excel_application` に共通化（再利用可能な関数を追加）。
- 解析雨量モードに `Excel候補日取込` ボタンを追加。  
  Excelシート名から候補日を抽出し、ユニーク日付で一括選択できるようにした。
- 開発者向けサブウィンドウ（開発者ツール）を追加。  
  CSV入出力系の補助機能を通常UIから分離して配置した。
- GUI起動時に開発者モードを引数で指定可能にした。  
  `cli gui --dev-mode 1` / `run_uc_rainfall_gui.py 1`

### Changed

- スタイルチューナーの再描画方式を改善。
  - Figure/Canvas の作り直しを減らし、再利用ベースに変更。
  - 描画キュー制御を見直し、連続更新時の取りこぼしと無駄描画を抑制。
- スタイルチューナーの操作UXを改善。
  - スライダーは確定操作（リリース）時に再描画。
  - 数値入力は `Enter` / フォーカスアウト時に反映。
  - 入力値に step 丸め・範囲クランプ・表示フォーマットを適用。
- スタイル調整項目を拡張。
  - `title_pad`（タイトル余白）
  - `y1_label_pad` / `y2_label_pad`（左右軸ラベル余白）
  - `y_tick_pad`（左右目盛余白）
- フォントサイズ調整の最小値を緩和。
  - タイトル: 4.0
  - 軸ラベル: 4.0
  - 目盛: 3.0
- スタイルチューナーの項目名・並び順を見直し、可読性を改善。
- 複数対象日の重複出力確認を、対象日ごとの確認から「全対象日を一括1回確認」へ変更。
- 実行中GUIログを強化し、進捗（`[i/N]`）や事前準備の状態が分かるようにした。
- 実行中は「処理を実行」ボタンを無効化し、重複実行を防止する挙動に変更。
- 解析雨量バッチ実行の事前計算共有を追加。
  - ZIP期間一覧の再走査を抑制
  - 流域定義読込の再実行を抑制
- ラスタ処理の最適化を追加。
  - 重み計算（overlap weights）の再利用
  - クリップ計画（window/mask）の再利用

### Fixed

- プレビュー縮小時に旧描画が残る問題を修正。  
  キャンバス実サイズ更新と中央再配置を毎回適用。
- スライダー操作時の不自然な挙動を緩和。  
  非確定時の不要再描画を抑制し、確定値ベースで安定化。
- 解析雨量モードの候補日選択反映が遅い問題を改善。  
  候補日選択時に不要なUI状態再設定を走らせないように変更。
- 一時展開ZIPのクリーンアップ時に `WinError 32` が発生する問題を緩和。  
  一時ディレクトリ削除にリトライ処理を導入。
- ラスタ最適化時に時系列データが固定化される不具合を修正。  
  データ配列の使い回しを廃止し、形状計算のみ再利用する方式へ変更。
