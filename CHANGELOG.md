# Changelog

このプロジェクトの変更履歴はこのファイルで管理します。  
形式は [Keep a Changelog](https://keepachangelog.com/ja/1.1.0/)、バージョニングは SemVer を前提にします。

## 運用ルール

- バージョンの基準は `pyproject.toml` の `[project].version`。
- 変更はまず `Unreleased` に追記。
- `pyproject.toml` の version を更新したタイミングで、`Unreleased` を同じバージョン見出しへ確定（リリース日付を付与）。
- セクションは原則 `Added` / `Changed` / `Fixed` を使用。

## [0.2.0] - 2026-03-18

### Added

- CLI に `excel-candidates` サブコマンドを追加。  
  入力Excelから候補日CSVを2点出力できるようにした。
  - `<Excel名>_候補日付リスト.csv`（日付・シート名・再分割フラグ）
  - `<Excel名>_候補日付一覧_unique.csv`（重複除去日付）
- Excel候補抽出ロジックを `excel_application` に共通化（再利用可能な関数を追加）。

### Changed

- スタイルチューナーの再描画方式を改善。
  - Figure/Canvas の作り直しを減らし、再利用ベースに変更。
  - 描画キュー制御を見直し、連続更新時の取りこぼしと無駄描画を抑制。
- スタイルチューナーの操作UXを改善。
  - スライダーは確定操作（リリース）時に再描画。
  - 数値入力は `Enter` / フォーカスアウト時に反映。
  - 入力値に step 丸め・範囲クランプ・表示フォーマットを適用。
- スタイル調整項目を拡張。
  - `title_pad`（タイトル余白）
  - `y1_label_pad` / `y2_label_pad`（左右軸ラベル余白）
  - `y_tick_pad`（左右目盛余白）
- フォントサイズ調整の最小値を緩和。
  - タイトル: 4.0
  - 軸ラベル: 4.0
  - 目盛: 3.0

### Fixed

- プレビュー縮小時に旧描画が残る問題を修正。  
  キャンバス実サイズ更新と中央再配置を毎回適用。
- スライダー操作時の不自然な挙動を緩和。  
  非確定時の不要再描画を抑制し、確定値ベースで安定化。
- 解析雨量モードの候補日選択反映が遅い問題を改善。  
  候補日選択時に不要なUI状態再設定を走らせないように変更。
